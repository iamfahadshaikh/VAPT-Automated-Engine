Advanced Security Scanner v2 — Usage (Kali Linux)

Modes
- gate (default): focused web checks (reachability, whatweb, sslscan, ffuf small, dalfox, xsstrike), lower noise.
- full: legacy broader sweep (more tools, slower), exploit tools optional via flag.

Summary
- Runs recon/vuln tooling with domain/subdomain handling (scans both if subdomain given).
- Produces per‑tool outputs plus executive, vulnerability, and remediation reports.

Requirements (Kali)
- Kali Linux with Python 3 installed (python3 and pip3 available).
- Network access and permission to scan the target.
- Recommended: install common tools via apt (see below) or use `--install-all`.

Quick Start (Kali terminal)
1) Navigate to the project folder (mounted if using WSL):
```bash
cd /path/to/project  # e.g. /mnt/c/Users/FahadShaikh/Desktop/something
```

2) Run a scan (replace TARGET):
```bash
# Gate mode (default, curated web checks)
python3 automation_scanner_v2.py TARGET --skip-install -p https

# Full mode (broader sweep)
python3 automation_scanner_v2.py TARGET --skip-install --mode full -p https
```

Examples
- Scan a subdomain (scans both subdomain and its root domain):
```bash
python3 automation_scanner_v2.py dev-erp.sisschools.org --skip-install -p https
```

- Scan a root domain only:
```bash
python3 automation_scanner_v2.py example.com --skip-install -p https
```

- Auto‑install missing tools first (non‑interactive):
```bash
python3 automation_scanner_v2.py example.com --install-all -p https
```

CLI Options
```text
usage: automation_scanner_v2.py [-h] [-p {http,https,both}] [-o OUTPUT]
																[--skip-install] [--install-all]
																target

positional arguments:
	target                Target domain or subdomain (e.g., example.com or dev.example.com)

optional arguments:
	-h, --help            Show this help message and exit
	-p, --protocol        http | https | both (default: https)
	-o, --output          Output directory name (auto‑generated if omitted)
	--skip-install        Skip prompts; run with currently installed tools
	--install-all         Auto‑install missing tools via apt/pip/go when possible
	--mode                gate | full (default: gate)
	--enable-exploit-tools Enable exploit-heavy tools (sqlmap/commix); off by default
```

Installing Common Tools on Kali
Many tools are preinstalled on Kali. If some are missing, install with apt:
```bash
sudo apt update
sudo apt install -y dnsrecon dnsutils dnsenum nmap traceroute whois testssl.sh sslscan openssl \
										wpscan whatweb golismero sqlmap xsser commix python3-pip
```
Extras (if you plan to use them and they’re not present):
```bash
# sslyze
pip3 install --break-system-packages sslyze

# findomain (Kali package or upstream release may vary)
sudo apt install -y findomain || true

# sublist3r (Subdomain)
sudo apt install -y sublist3r || pip3 install --break-system-packages sublist3r

# assetfinder / dalfox / corsy (often installed via go/pip or manual)
# Install only if needed; the scanner will still run without them.
```

What the Scanner Does
- DNS & Subdomain: A/AAAA/MX/NS/TXT/CAA/ANY queries, zone‑transfer attempts, subdomain enumeration.
- Network: Fast/service/version scans (nmap), ping, traceroute.
- SSL/TLS: Protocols, ciphers, certificate info (testssl, sslscan, sslyze, openssl).
- Web App: Tech fingerprinting, light crawling/fuzzing (whatweb, wapiti, ffuf, wpscan where applicable).
- Vulnerability: Lightweight checks for XSS/SQLi/injections (xsstrike/sqlmap/commix/xsser/etc.).

Output
- A timestamped directory is created, for example:
	scan_results_<target>_<YYYYMMDD_HHMMSS>/
	- Per‑tool outputs (e.g., testssl_full.txt, nmap_sV.txt, host_a_<target>.txt)
	- EXECUTIVE_SUMMARY.txt
	- vulnerability_report.json
	- remediation_report.json

Subdomain Handling
- If you pass a subdomain (e.g., dev-erp.sisschools.org), the scanner also extracts and scans the root domain (sisschools.org).
- Outputs for subdomain and root domain are saved as separate files within the same results folder.

Troubleshooting
- Missing tools: re‑run with `--install-all` or install the specific tool via apt/pip/go.
- Long runtime: SSL checks and nmap script scans can be slow; start with `-p https` and avoid heavy flags for faster runs.
- Permissions/Network: ensure your Kali host has outbound network access and you have authorization to test.

Ethical Notice
- Only scan systems you own or have explicit permission to test.

Common One‑Liners (Kali)
```bash
# HTTPS‑only scan, skip installs
python3 automation_scanner_v2.py TARGET --skip-install -p https

# Auto‑install missing tools, then scan
python3 automation_scanner_v2.py TARGET --install-all -p https
```
